# Copilot Instructions - Gestão de TCCs

Você é um assistente de codificação especializado neste projeto Django + React TypeScript. Siga rigorosamente estas instruções.

## 🎯 Objetivo do Projeto

Sistema de gestão de Trabalhos de Conclusão de Curso (TCCs) para UFLA com:

- **Backend**: Django REST Framework (Python)
- **Frontend**: React 19 + TypeScript + Vite + Material-UI
- **Database**: PostgreSQL
- **Deploy**: Docker

## 📋 Regras Gerais

### 1. **Nunca Modifique Lógica Existente**

- Apenas adicione funcionalidades
- Se precisar alterar código existente, faça com extremo cuidado
- Sempre preserve a intenção original do código

### 2. **TypeScript Obrigatório**

- Tipagem explícita em TUDO
- Sem `any`, `unknown` deve ser tratado
- Interfaces para componentes, tipos para simples

### 3. **Testes para Novas Funcionalidades**

- Sempre escreva testes ao lado do código
- Use Vitest + @testing-library
- Mire por cobertura > 80%

### 4. **Documentação com JSDoc**

- Componentes públicos precisam de JSDoc
- Funções utilitárias precisam de documentação
- Exemplos de uso em @example

```typescript

  const [user, setUser] = useState<User | null>(null);

    fetchUser(id).then(setUser);
  }, [id]); // Dependencies explícitas

  return user;
}
```

## 📚 Quando Adicionar Código

### ✅ Adicione

1. **Testes** - Sempre que adiciona uma função
2. **Documentação** - JSDoc para funções públicas
3. **Tipos** - Tipagem explícita
4. **Validação** - Use `validators.ts`
5. **Constantes** - Em `constants.ts`, não hardcode
6. **Tratamento de Erro** - Use `errorHandler.ts`

### ❌ Não Adicione

1. `any` - Use `unknown` e trate
2. Código sem testes - Exceto componentes simples de UI
3. Hardcoded strings/números - Use constantes
4. `console.log` em produção
5. Deps em runtime sem justificar - Pergunte primeiro
6. Lógica em componentes - Mova para hooks/utils
